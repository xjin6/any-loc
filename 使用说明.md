# AnyLoc 使用说明（超简单版）

AnyLoc 现在 **Windows 和 macOS 双端都能用**。下面分开讲。

---

## 🪟 Windows 用户

### 现在怎么用？

**桌面上有个 `AnyLoc` 图标 → 双击它就行。**
（或者进 `AnyLoc` 文件夹，双击里面的 `AnyLoc.exe`）

### 双击后会发生什么

1. **弹出「用户账户控制 / UAC」窗口** → 点**「是」**。
   （必须，连 iPhone 的隧道需要管理员权限。以前那两个 .bat、右键管理员——现在这一个就全搞定。）
2. 弹出一个黑色小窗口（正常，**别关**，可最小化）。
3. 几秒后**浏览器自动打开深色地图页面**。
   - 没自动开就手动打开浏览器，地址：`http://127.0.0.1:8765`

### 第一次用之前，iPhone 要准备好（只需一次）

1. **电脑装了 iTunes**（提供 Windows 识别 iPhone 的驱动）。
2. **iPhone 开「开发者模式」**：设置 → 隐私与安全性 → 拉到底 → 开发者模式 → 打开 → 重启手机。
   - ⚠️ 找不到这一项？先用数据线插电脑、点一次「信任」，它就会出现。
3. **数据线把 iPhone 插电脑**，解锁，弹「信任这台电脑吗」→ 点**信任**。

用完直接**关掉那个黑窗口**就全部停止了。

## 🍎 macOS 用户

### 怎么安装

**双击 `AnyLoc.pkg` 安装包** → 一路下一步 → AnyLoc 就装进了「应用程序」。
（或者直接把 `AnyLoc.app` 拖进「应用程序」文件夹。）

> ⚠️ 这个 `.pkg` / `.app` 是 **ad-hoc 签名、未做苹果公证**（没有付费的开发者证书）。
> 从网络/AirDrop 传过来后，macOS 首次打开可能会拦截并提示「无法验证开发者」。
> 解决：**右键点它 →「打开」→ 再点「打开」**（只需一次）；或在终端执行
> `xattr -dr com.apple.quarantine AnyLoc.pkg`（对 `.app` 同理）清掉隔离标记。
>
> 另外：打包出来的 App **架构跟打包时用的 Mac 一致**——在 Apple 芯片（M 系列）上打的包
> 只能在 Apple 芯片 Mac 上跑，Intel Mac 需要在 Intel 上打包（或用 universal2）。
> 直接从源码运行则不挑架构。

### 怎么用

1. 在**启动台 / 应用程序 / 聚焦搜索**里打开 **AnyLoc**。
   - ⚠️ 第一次打开若提示「无法打开，来自身份不明的开发者」：
     **右键 AnyLoc.app → 打开 → 再点「打开」**，只需这一次。
2. **会弹出一个「终端」窗口，让你输入开机密码**（这是 `sudo`，连 iPhone 的隧道需要
   root 权限，相当于 Windows 的管理员）。输入密码回车即可。
   - 输密码时**屏幕不显示任何字符**，是正常的，输完直接回车。
   - macOS 可能会弹一次「AnyLoc 想要控制“终端”」→ 点**「好」/「OK」**即可。
3. 几秒后**浏览器自动打开深色地图页面** `http://127.0.0.1:8765`。
4. 后面操作和 Windows 完全一样（见下面「在网页里改定位」）。
5. 用完想停 → **直接关掉那个终端窗口**。

### 第一次用之前，iPhone 要准备好（只需一次）

macOS **不用装 iTunes**（系统自带识别 iPhone 的驱动），只需：

1. **iPhone 开「开发者模式」**：设置 → 隐私与安全性 → 拉到底 → 开发者模式 → 打开 → 重启手机。
   - ⚠️ 找不到这一项？先用数据线插电脑、点一次「信任」，它就会出现。
2. **数据线把 iPhone 插电脑**，解锁，弹「信任这台电脑吗」→ 点**信任**。

---

## 在网页里改定位（两端相同）

1. 点右边的 **Connect（连接）** 按钮。
2. 左上角小圆点**变绿** = 手机连上了。
3. **在地图上点任意位置** → iPhone 定位瞬间跳过去。
4. 想「走路移动」→ 拖右边的**摇杆**，或用键盘 **W/A/S/D**
   （可选 走路/跑步/骑车/开车 速度，按住 Shift 加速 3 倍）。
5. 不想改了 → 点 **Reset GPS（恢复真实定位）**。

---

## 连不上 / 出错怎么办

- 确认程序是**带权限**跑的：
  - Windows：双击 exe 后**点了 UAC「是」**；
  - macOS：**在终端里输了开机密码**，且那个终端窗口还开着。
- **解锁手机**，重新点一次「信任」。
- 确认 iPhone **开发者模式**开着。
- 拔了数据线重插，或换一个 USB 口（直接插机身/机箱上的口，比插 USB hub 更稳）。
- 第一次连接会自动**下载并挂载「开发者镜像」**，需要联网，可能等 1 分钟。

---

## 有什么限制

- 目标是 **iOS 17 / 18+**，靠「隧道」工作 → 那个黑窗口 / 终端窗口要一直开着。
- 需要**数据线**连着（纯无线也能做，但要额外设置，先没做，保证简单可靠）。
- 定位造假有正当用途（开发、测试、隐私）。拿去在游戏/打卡/社交里作弊可能违反对方规则、
  甚至法律，后果自负。

## 给开发者：怎么重新打包

改了 `backend.py` / `launcher.py` / `web/` 之后：

### Windows → 生成 `AnyLoc.exe`
```powershell
# 测试版（无 UAC，可直接 --selftest 验证）
$env:ANYLOC_VARIANT="test";  pyinstaller --clean --noconfirm AnyLoc.spec
dist\AnyLocTest\AnyLocTest.exe --selftest      # 应输出 RESULT: PASS

# 正式版（带管理员权限清单）
$env:ANYLOC_VARIANT="final"; pyinstaller --clean --noconfirm AnyLoc.spec
# 产物在 dist\AnyLoc\，整个文件夹拷走即可运行
```

### macOS → 生成 `AnyLoc.app` + `AnyLoc.pkg`
```bash
./build-mac.sh          # 正式版：dist/AnyLoc.app + dist/AnyLoc-1.0.0.pkg
./build-mac.sh test     # 测试版：dist/AnyLocTest.app
# 验证测试版（无需 root）：
dist/AnyLocTest.app/Contents/MacOS/AnyLocTest --selftest   # 应输出 RESULT: PASS
```
`build-mac.sh` 会自动打包、做 ad-hoc 代码签名、再生成能装进「应用程序」的 `.pkg`。

### 只调界面（不用打包，两端都行）
改 `web/` 里的文件，浏览器 ~1 秒自动刷新：
- Windows：双击 `dev.bat`（仅界面），或右键「以管理员身份运行」（带真机隧道）。
- macOS：`./dev.command`（仅界面），或 `sudo ./dev.command`（带真机隧道）。

> 注意：本项目在 OneDrive 同步目录里，打包时 OneDrive 偶尔会锁文件导致
> `Access is denied` / 文件占用。重试一次通常即可，或先暂停 OneDrive 同步再打包。
